## 一、设计总览

### 1.1 设计理念

**对话即一切** — 用户只面对一个 AI 对话窗口，所有数据展示以"卡片"形式嵌入对话流中。点击卡片全屏展开查看详情。没有复杂的导航层级，没有需要学习的操作流程。

### 1.2 技术栈

- 框架：Expo (React Native)，一套代码 iOS + Android
- 状态管理：Zustand
- 图表库：victory-native / react-native-chart-kit
- 动画：react-native-reanimated
- 数据层：@supabase/supabase-js
- 路由：Expo Router（文件路由）
- 媒体处理：expo-image-picker / expo-av

### 1.3 项目结构

```

│   ├── (auth)/             # 登录/注册
│   ├── (main)/

│   │   └── settings/       # 设置页（右上角齿轮进入）
│   └── _layout.tsx
```

### 1.6 实时通信：Streamable HTTP

对话数据流采用 Streamable HTTP：

1. 用户发消息 → `ai-chat.ts` 通过 HTTP POST 请求 orchestrator Edge Function
2. Orchestrator 识别意图 → 调用对应业务 Edge Function
3. 业务函数响应以 chunked transfer 流式返回文本 + 卡片 JSON，前端用 `fetch` + `ReadableStream` 读取
4. `StreamingText` 组件逐字渲染，卡片 JSON 触发 Card 组件
5. 业务函数同时写入 Supabase + 触发 snapshot-refresh

## 二、卡片组件系统

### 2.1 卡片双态设计

所有数据展示通过卡片承载，卡片有两种状态：

**缩略态（嵌入对话流）：**
- 全宽独立展示（左边缘与AI气泡对齐，max-width 与气泡一致），不包裹在气泡内，高度自适应但有最大限制
- 圆角 16px，轻阴影（`0 2px 8px rgba(0,0,0,0.08)`）
- 底部有"点击查看详情"提示 + 向上箭头图标
- 数据量少时自动缩小，数据量多时截断 + "查看更多"

**全屏态（点击展开）：**
- 从底部滑入的全屏面板（iOS Sheet 风格，react-native-reanimated 实现）
- 支持手势下拉关闭（带阻尼效果）
- 图表支持双指缩放、左右滑动切换时间段
- 顶部有分享按钮（截图分享到微信）
- 统一时间导航：左右滑动切换时间段 + 时间范围选择器

### 2.2 动态时间范围

卡片不绑定固定时间段，由 Agent 根据用户意图传入参数：

```
用户："看看这周训练情况" → TrendChart(range='7d')
用户："最近一个月的趋势" → TrendChart(range='30d')
用户："明天练什么" → TrainingCard(date='2026-02-20')
用户："下周排班" → ScheduleCard(range='next_week')
用户："上月和这月对比" → RadarChart(date='2026-02', compare_date='2026-01')
```

缩略态自适应展示对应时间范围的数据，全屏态支持进一步调整时间范围。

### 2.3 卡片渲染协议

Agent 返回的消息中，卡片以结构化 JSON 嵌入：

```json
{
  "type": "trend_chart",
  "title": "最近30天训练趋势",
  "time_range": {"start": "2026-01-20", "end": "2026-02-19"},
  "data_source": "training_sessions",
  "metrics": ["completion_rate", "success_rate"],
  "child_id": "xxx"
}
```

前端拿到 JSON 后，通过 `@supabase/supabase-js` 查询对应时间范围的数据，渲染成对应的卡片组件。缩略态和全屏态共用同一份数据，只是渲染密度不同。

## 三、各角色详细设计

### 3.1 家长端

#### 视觉风格

**色彩：** 详见 `06-前端交互与体验文档.md` §1.2 家长端配色体系。主色教育紫 `#7C4DFF`，背景淡紫白 `#F5F3FF`。

**特色元素：**
- AI 头像：3D/2.5D 卡通星星形象（NanoBanana 生成，呼应"星途"品牌）
- 情绪日历用表情符号（😊😐😢😡）而不是数字
- 训练完成后有小星星收集动画
- 里程碑达成时星星头像变成庆祝动画 + 彩带飘落

#### 对话页面
1. 用户打开 App → 进入对话页
2. AI 根据时间和上下文主动问候 + 推荐今日行动
3. 用户通过对话获取所有信息和服务
4. 数据以卡片形式嵌入对话流
5. 点击卡片全屏查看详情

输入区域（蚂蚁阿福风格卡片式输入框）：
- 整体：圆角 16px 白色卡片容器，聚焦时边框高亮 + 阴影
- 顶部行：左侧语音输入按钮（圆形）+ 中间文字输入区（textarea，聚焦时自动增高）+ 右侧"+"按钮（拍照/相册）
- 底部行（聚焦时展开）：左侧"乐乐 ⇄"孩子切换按钮（点击弹出切换弹窗）+ 右侧"查看档案 ›"按钮
- 外部右侧：发送按钮（渐变圆形）
- 快捷操作气泡（输入框上方，根据上下文动态出现）：
  - 早上打开："开始今天的训练" "查看看板"
  - 训练结束后："记录训练" "跳过今天"
  - 收到通知后："查看详情" "稍后再看"

#### 媒体消息组件

对话流中支持 5 种消息类型：文字、图片、视频、语音、卡片。媒体消息的前端组件和数据流如下：

**组件目录：**
```
src/components/chat/
├── MessageBubble/        # 文字消息气泡（Markdown 渲染）
├── ImageMessage/         # 图片消息（缩略图 + 全屏查看）
├── VideoMessage/         # 视频消息（封面 + 播放器）
├── VoiceMessage/         # 语音消息（声波条 + 播放）
├── CardMessage/          # 卡片消息（缩略态 + 全屏展开）
├── StreamingText/        # 流式文本逐字渲染
├── VoiceInput/           # 语音输入录制
└── QuickActions/         # 快捷操作横滑条
```

**ImageMessage 组件：**
- 依赖：`expo-image-picker`（选图/拍照）、`react-native-image-viewing`（全屏查看）
- 上传：`Supabase Storage` → `chat-media/{child_id}/{timestamp}.jpg`
- 缩略态：最大宽度 65%，圆角 12px，点击全屏（支持双指缩放）
- 加载态：灰色骨架屏 + 渐变闪烁
- 失败态：断裂图片图标 + "点击重试"

**VideoMessage 组件：**
- 依赖：`expo-image-picker`（录制/选择）、`expo-av`（播放）
- 上传：`Supabase Storage` → `chat-media/{child_id}/{timestamp}.mp4`，分片上传
- 封面图：视频第一帧自动提取，`brightness(0.85)` 暗化
- 播放按钮：居中 44×44px 圆形毛玻璃按钮
- 时长标签：右下角 `rgba(0,0,0,0.6)` 背景
- 限制：Phase 1 最大 60秒 / 50MB

**VoiceMessage 组件：**
- 依赖：`expo-av`（录制/播放）
- 声波条：12-16 根竖条，宽 3px，高度随音量变化，颜色跟随角色主色
- 播放动画：声波条从左到右依次高亮
- 时长显示：右侧 `M:SS` 格式
- 限制：最大 60秒

**媒体上传服务（`services/media.ts`）：**
```typescript
// 核心接口
uploadImage(uri: string, childId: string): Promise<string>  // 返回 public URL
uploadVideo(uri: string, childId: string): Promise<string>
uploadVoice(uri: string, childId: string): Promise<string>
generateThumbnail(videoUri: string): Promise<string>        // 视频封面提取
```

#### 卡片在对话流中的展示

卡片以缩略态嵌入对话流，不包裹在气泡内，直接作为独立消息展示：

**渲染流程：**
1. AI 流式回复中返回 `event: cards` 事件，携带卡片 JSON
2. 前端 `chatStore.finishAssistantMessage()` 将 cards 附加到消息
3. `MessageBubble` 检测到 `message.cards` 后，渲染对应的卡片组件
4. 卡片组件根据 `card_type` 分发到对应的卡片组件
5. 卡片内的数据通过 `data_source` 参数，用 Supabase SDK 查询渲染

**卡片操作按钮：**
- 缩略态最多 2 个按钮（primary + secondary），水平等宽排列
- 全屏态最多 4 个按钮
- 按钮点击后发送预设消息到对话流或调用对应 Edge Function

**图表渲染：**
- 缩略态：SVG 内嵌渲染（轻量快速，跟随角色主色）
- 全屏态：`victory-native` 渲染（支持缩放、滑动、tooltip 交互）

#### 右上角胶囊菜单

小程序风格胶囊按钮（左侧三点菜单 + 分隔线 + 右侧关闭按钮），点击三点弹出下拉菜单：
- 设置页（8个子页面：修改昵称、修改手机号、使用偏好、数据访问权限、用户协议、隐私政策、关于星途、意见反馈）
- 通知（推送偏好）
- 升级VIP
- 注销账号

#### 设置页（04-settings）

从右上角胶囊菜单进入，slide-in 动画。主列表包含以下功能组：

**账号信息组：**
- 修改昵称：点击进入子页面，输入框编辑
- 修改手机号：点击进入子页面，短信验证码验证

**偏好设置组：**
- 使用偏好：训练提醒开关+时间、AI回复风格（简洁/详细/温暖）、语言
- 数据访问权限：顶部孩子切换Tab（多孩家庭），每个孩子独立的医生/老师权限开关

**法律信息组：**
- 用户协议：完整协议内容，与登录页一致
- 隐私政策：完整隐私政策内容，与登录页一致

**其他组：**
- 关于星途：版本号、官网、客服
- 意见反馈：文字反馈提交
- 升级VIP（渐变按钮）
- 注销账号（红色文字按钮，需二次确认）

#### 对话页左上角快捷菜单

**设计理念：** 对话页是主战场，但用户有时需要快速浏览结构化数据（训练记录、评估报告、情绪日历等），不想每次都通过对话获取。对话页左上角提供一个菜单按钮，点击弹出宫格快捷入口面板，直接跳转到对应的全屏卡片视图。

**触发方式：** 对话页顶部导航栏左侧的菜单图标（Lucide: `layout-grid`），点击后从左侧滑入半屏面板（或底部 Sheet 弹出）。

## 五、对话交互设计

### 5.1 消息渲染流程

1. Streamable HTTP 流式文本逐字渲染（`StreamingText` 组件，react-native-reanimated 驱动）
2. 对话气泡内支持 Markdown 表格渲染，用于展示结构化数据（如各项训练的完成率、较上周对比、时长等）
3. 卡片在文字完成后 0.3s 从底部滑入
4. 多张卡片依次滑入，间隔 0.2s
5. 用户可以在任何时候点击卡片展开全屏

### 5.2 卡片交互手势

| 手势 | 行为 |
|------|------|
| 点击卡片 | 全屏展开（从底部滑入，带弹性动画） |
| 全屏态下拉 | 关闭（带阻尼效果） |
| 全屏态左右滑动 | 切换时间段 |
| 全屏态双指缩放 | 调整图表粒度（日→周→月） |
| 长按消息 | 引用回复 |

### 5.3 快捷操作气泡

输入框上方根据上下文动态出现快捷操作：

| 上下文 | 快捷操作 |
|--------|---------|
| 早上首次打开 | "开始今天的训练" "查看看板" |
| 训练结束后 | "记录训练" "跳过今天" |
| 收到通知后 | "查看详情" "稍后再看" |
| 评估完成后 | "查看结果" "分享给医生" |
| 空闲时 | "记录生活" "查看进展" |

### 5.4 用户输入方式

- **文字输入**：底部固定输入框，支持多行
- **语音输入**：长按麦克风按钮录音，松开自动转文字发送
- **图片/视频**：点击"+"按钮，选择拍照/相册/录视频
- **快捷操作**：点击输入框上方的气泡按钮

## 七、全屏态时间导航

### 7.1 时间选择器
